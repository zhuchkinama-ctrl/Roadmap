# Цикл обратной связи в PCAM
## 1 Промпт-правило для LLM
```xml
<feedback_loop priority="CRITICAL">
<on_error>
После каждой ошибки ты ОБЯЗАН:
1 Сгенерировать DIAGNOSTIC_REPORT в JSON-формате
2 Описать root cause и примененное решение
3 Предложить новое правило для предотвращения повторения
4 Сохранить отчет в logs/incidents/
5 Обновить rules/self_healing.xml новым паттерном
</on_error>
<report_structure>
{
"incident": "краткое описание ошибки",
"root_cause": "глубинная причина",
"fix_applied": "что конкретно исправлено",
"rule_proposal": "предложение нового правила",
"prevention": "как избежать в будущем"
}
</report_structure>
<forbid>
❌ Молча исправлять ошибки без документирования
❌ Фиксить симптом вместо причины
❌ Пропускать обновление rules после инцидента
</forbid>
</feedback_loop>
