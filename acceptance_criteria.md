# Критерии приёмки Liquibase схем

## 1. Успешный запуск миграций
- Выполнить `mvn liquibase:update`.
- В базе данных должны появиться таблицы `users`, `tracks`, `notes`, `track_permissions` с указанными колонками и ограничениями.

## 2. Откат последнего changeset
- Выполнить `mvn liquibase:rollback -Dliquibase.rollbackCount=1`.
- Последняя созданная таблица (`track_permissions`) должна быть удалена, остальные таблицы оставаться.

## 3. Повторный запуск без ошибок
- После отката выполнить `mvn liquibase:update` повторно.
- Все таблицы должны создаваться без конфликтов (например, уникальные ограничения).

## 4. Интеграция в CI/CD
- В CI‑pipeline добавить шаг `mvn liquibase:update` против тестовой БД.
- Сборка должна проходить успешно, миграции применяться и откатываться при необходимости.
